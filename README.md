## 主要なポイント（運用上の基準）

| 項目 | 値 |
|---|---|
| **障害レベル定義** | L3 / L2 / L1（速報期限・エスカレ期限・復旧目標を分離）→ 第5章で正式定義 |
| **L3 速報期限** | 検知後 15 分以内 |
| **L3 エスカレ期限** | 30 分以内 |
| **L3 復旧目標** | 同一リージョン 4 時間／DR 移行時 24 時間以内 |
| **RTO（復旧目標時間）** | 同一リージョン 2 時間／DR（東京→大阪）3 時間 |
| **RPO（復旧目標時点）** | 24 時間（RDS 自動スナップショット 0:00 取得） |
| **ログ保持** | CloudWatch 30 日 → S3 7 年 → Glacier 最大 15 年 |
| **データ保持** | RDS 本体: 恒久保存（暫定）／バックアップ S3: 7 年 |
| **年次復元テスト** | 毎年 12 月 |
| **使用ツール** | Trello / メール / Teams / GitLab（Slack は使用しない） |

## ドキュメントの読み方

1. 初めての方は **本文章/ETRCARS_内部統制書.md** を最初に読んでください
2. その後、関心ある章（第1〜11章）または手順書を参照
3. **章間で参照がある場合**、リンク先を辿ってください（例: 第2章の障害レベルは第5章を参照）
4. 別紙の **xlsx は実際の運用記入帳票** です。Web 上では Excel で開く必要があります

## 機密保持

- このリポジトリは **プライベート**設定です
