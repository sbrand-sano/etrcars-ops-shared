## AWS 暗号化手順ドキュメント

目的
個人データ暗号化キーを安全に管理、保存、およびローテーションするためのプロセスを定義することです。これにより、セキュリティおよびプライバシー基準への準拠を維持しながら、機密データが不正アクセスから保護されることが保証されます。
範囲
この手順は、AWS 環境内で個人データや機密データを処理するすべてのシステムとサービスに適用されます。具体的には、次のとおりです。
AWS Secrets Manager - 暗号化キーを安全に保存します。
Amazon ECS (Elastic Container Service) – 暗号化キーを使用するアプリケーションをデプロイします。
手順
AWS Secrets Managerに個人データ暗号化キーを保存する
任意のプログラミング言語を使用して、新しい32バイトの16進文字列暗号化キーを生成します。
シークレットをインポートする
secrets.token_hex( 32 )

AWS マネジメントコンソールにログインします。
Secrets Managerに移動する → ETRCARS_HASH_KEYを選択します。
ETRCARS_HASH_KEY の秘密の値。
キー- HASHY_KEY
値- 手順 1 の暗号化キーを貼り付けて追加します。
保存
ECS タスク実行ロールと Secrets Manager 間の信頼関係を構成する
IAM コンソールを開き、ECSタスク実行ロールを見つけます。
ECS タスク実行ロールに Secret Manager の値を読み取る権限があることを確認します。
アプリケーションを再デプロイする
Gitlab CI/CD パイプラインを使用してアプリケーションを再デプロイします。
新しいタスク定義が ECS に登録されていることを確認します。
ECS へのデプロイが成功したら、顧客データがまだ正しく表示されているかどうかを確認します。
