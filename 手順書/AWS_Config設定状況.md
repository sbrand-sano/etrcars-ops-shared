以下は、リストの詳細な説明と、それらがまだ非準拠である理由です。 

1. 

説明: Amazon VPC の EC2 VPC BPA オプションモードがインターネットトラフィックをブロックするように設定されているかどうかを確認します。EC2 VPC BPA オプションモードがトラフィックをブロックするように設定されていない場合、ルールは NON_COMPLIANT になります。オプションで、ルールはトラフィックがブロックされる方向をチェックできます。

非準拠の理由: インターネットからの接続を必要とするパブリックサブネットで ALB を使用しています。また、外部インターネット接続にはインターネットゲートウェイと VPC エンドポイントを使用しているため、インターネットゲートウェイをブロックできません。 

2.
説明:プライベートAmazon Elastic Container Registry (ECR)リポジトリでタグの不変性が有効になっているかどうかを確認します。プライベートECRリポジトリでタグの不変性が有効になっていない場合、このルールはNON_COMPLIANTとなります。
非準拠の理由: 現在、ECS イメージ設定は変更可能です。これを不変に変更すると、本番サーバーへのデプロイメントが機能しなくなり、デプロイメント パイプライン構成の変更が必要になります。これは、実際のアプリケーション デプロイメント プロセスに影響する可能性があります。 

3. 

説明: Amazon Elastic Container Service (Amazon ECS) コンテナがルート ファイルシステムに対して読み取り専用アクセス権のみを持っているかどうかを確認します。ECSTaskDefinitions のコンテナ定義で readonlyRootFilesystem パラメータが 'false' に設定されている場合、ルールは NON_COMPLIANT です。

非準拠の理由: ECS の readonlyRootFilesystem を変更すると、現在のデプロイメント プロセスでは機能しなくなります。ECS コンテナのボリューム マウント構成を変更する必要があり、実行中のアプリケーションに影響します。 

4. 

説明: 暗号化により、EC2 インスタンスに接続された EBS ボリュームを保護します。

非準拠の理由：EC2 で現在使用されているボリュームを暗号化対象に変更することはできません。新しい EC2 ボリュームを再作成し、EC2 に再アタッチする必要がありますが、これによりセッションマネージャーと EC2、RDS データベース接続間の現在の接続構成に影響する可能性があります。5 

.
説明: Amazon Relational Database Service (Amazon RDS) データベースの管理者ユーザー名がデフォルト値から変更されているかどうかを確認します。このルールは RDS データベースインスタンスでのみ実行されます。管理者ユーザー名がデフォルト値に設定されている場合、ルールは NON_COMPLIANT となります。

非準拠の理由: 現在の RDS 管理者設定を変更することはできません。これは、実行中のアプリケーションに影響し、アプリケーションとデータベースの接続にも影響する可能性があるためです。
6.
説明: このコントロールは、RDSインスタンスがデータベースエンジンのデフォルトポートを使用しているかどうかを確認します。

非準拠の理由: 現在のRDSインスタンスのポート設定を変更すると、実行中のアプリケーションに影響し、データベース接続に問題が発生する可能性があります。7 

. 

説明: AWS Secrets Managerのシークレットのローテーションが有効になっているかどうかを確認します。

非準拠の理由: 現在、シークレットマネージャーのローテーションは手動で行われており、自動ローテーション設定は必要ありません。
8.

非準拠の理由: 当社にはシークレットのローテーション期間に関する独自のルールがあります。
