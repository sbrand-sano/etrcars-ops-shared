ALBメンテナンス手順
初期アクセスとステータスチェック
1. IAM ユーザーで AWS マネジメントコンソールにログインします。
2. EC2 → ロードバランサー → `etrcars-ecs-alb` に移動します。
3. ALBのステータスを確認する
- 詳細タブ→状態（アクティブになっていることを確認してください）
- [監視] タブで、メトリック (リクエスト数、HTTP エラーなど) を確認します。
ヘルスチェック検証
1. ターゲットグループのステータスを確認する
- EC2 → ターゲットグループ → `etrcars-ecs-target-group` に移動します
- ターゲットタブ→登録されたターゲットのヘルスステータスを確認する
- ターゲットが正常であることを確認します（現在 3 つのターゲットに接続されています）
リスナーとルールの構成チェック
1. リスナーの設定を確認する
- 「リスナーとルール」タブで、次の点を確認します。
- プロトコル: `HTTPS`
- ポート: `443`
- デフォルトのアクション: `etrcars-ecs-target-group` に転送
- プロトコル: `HTTP`
- ポート: `80`
- デフォルトのアクション: `etrcars-ecs-target-group` に転送
セキュリティ構成チェック
1. セキュリティグループの設定を確認する
- セキュリティタブ→セキュリティグループIDをクリック
- セキュリティグループ名が `etrcars-alb-sg` であることを確認します
回復方法
上記のシナリオを確認した後で何か問題がある場合は、次の方法で ALB を回復できます。
1. 新しいロードバランサーを作成する
EC2 → ロードバランサー → ロードバランサーの作成に移動します
「Application Load Balancer」を選択し、「作成」をクリックします。
基本設定を構成します。
- ロードバランサー名を設定する
- VPCを古いものと同じ設定にする
- サブネットを古いものに設定する
- セキュリティグループを古いものに設定する
- ターゲットグループを `etrcars-ecs-target-group` に設定します
- その他（以前と同じ）
Cloudflareで新しいDNSを更新する
回復後
ALB を回復した後、次の点を確認します。
1. システムが正常に動作していることを確認する
- ウェブサイトをチェックして、データが正しく表示されているか確認します
- iPadの通常の操作機能が正常に動作しているかどうかを確認します
2. ALBメトリクスを確認する
- EC2 → ロードバランサー → `etrcars-ecs-alb` → モニタリングタブに移動します
- リクエスト数が正常であることを確認します
- HTTP 5XX エラーが最小限またはゼロであることを確認します
- レイテンシが許容範囲内であることを確認します
3. ターゲットグループの健全性を確認する
- EC2 → ターゲットグループ → `etrcars-ecs-target-group` → ターゲットタブに移動します
- すべてのターゲットが正常であることを確認する
