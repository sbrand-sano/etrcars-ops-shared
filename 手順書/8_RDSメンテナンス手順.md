RDS メンテナンス手順
- IAM ユーザーでAWS マネジメントコンソールにログインします。
- Aurora および RDS → データベース → etrcarsに移動します。
- RDS のステータスを確認します
-概要タブの下 -> ステータス ( 「利用可能」と表示されていることを確認してください)
- ログとイベントを確認します
- [ログとイベント]タブの下-> Cloudwatchアラーム (CPU と RAM の両方が正常であることを確認します)
- 最近のイベントを確認します
-ログとイベントタブの下 ->最近のイベント (疑わしいイベントがないことを確認します)
- データベースの負荷を確認します
- 概要タブの下 -> 現在のアクティビティ (赤い線を超えていないことを確認します)
- データベースストレージを確認する
-監視タブの下 -> FreeStorageSpace (ストレージが残っていることを確認する)
- 接続用の VPC セキュリティ グループを確認します
- [接続とセキュリティ]タブ -> [セキュリティ] -> [VPC セキュリティ グループ] -> [etrcars - rds -sg]を選択-> [受信ルール] (ポートが適切に設定されていることを確認します) 
- [EC2 サービス] -> [セキュリティ] グループ-> etrcars - ecs -sg を選択 -> 送信ルール (ポートが RDS に接続できるように適切に構成されていることを確認してください)
回復方法
上記のシナリオを確認した後に何か問題が発生した場合、RDS を回復するために試すことができる方法は次のとおりです。
- Aurora および RDS → データベース → etrcarsに移動します。
- RDS を再起動
- [アクション] -> [再起動] をクリックします。
- スナップショットからの復元
- 「メンテナンスとバックアップ」タブ -> 「スナップショット」 -> 最新のスナップショットを復元
- ポイントインタイムリカバリ（データ破損の場合） 
- 「アクション」をクリック -> 「ポイントインタイムに復元」 -> 復元したいデータと時間を選択
- ストレージを増やす（ストレージがいっぱいの場合） 
- 「変更」をクリック -> 「ストレージ」タブで、増やしたい量を入力します
リカバリ後
データベースをリカバリした後、以下の点を確認する
必要があります。
- Aurora および RDS → データベース → etrcarsに移動します。
- RDS エンドポイント URL を確認します
- [接続とセキュリティ]タブ-> [エンドポイント] (ECS が正しいエンドポイントURLを使用していることを確認します)
- システムが正常に動作していることを確認します
- ウェブサイトをチェックして、データが正しく表示されているかどうかを確認します - iPad をチェックして、通常の操作機能が正常に動作しているかどうかを確認します
