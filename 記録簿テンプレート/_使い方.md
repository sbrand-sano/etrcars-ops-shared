# 実施記録（records/）

ETRCARS 運用保守の実施記録を、種別ごとに格納する場所。
CODIUM 監査向けの報告は、ここの記録を集約して `records/MONTHLY/YYYY-MM.md` で送付する。

## ディレクトリ

| パス | 種別 | テンプレート |
|---|---|---|
| `patch/YYYY/MM/` | パッチ適用 | [patch_report.md](../templates/patch_report.md) |
| `log/YYYY/MM/` | ログチェック | [log_check_report.md](../templates/log_check_report.md) |
| `audit/` | 現地監査・世代交代 | [audit_report.md](../templates/audit_report.md) |
| `vulnerability/` | 脆弱性対応 | [vulnerability_report.md](../templates/vulnerability_report.md) |
| `account/` | アカウント申請・使用履歴 | [account_request.md](../templates/account_request.md), [account_usage_log.md](../templates/account_usage_log.md) |
| `cloudflare/` | Cloudflare 設定変更 | [cloudflare_change.md](../templates/cloudflare_change.md) |
| `incident/` | 障害報告 | [incident_report.md](../templates/incident_report.md) |
| `MONTHLY/` | 月次ダイジェスト | [monthly_digest.md](../templates/monthly_digest.md) |

## 記録の作り方

1. 該当ディレクトリの直下に新しい MD を作る（命名規則は CONTRIBUTING.md 参照）
2. テンプレートをコピーして埋める
3. 関連する raw データ（xlsx / pdf / png）は同階層に置く
4. フロントマターは必ず記入（後で集計可能にするため）
5. Git コミット: `record: YYYY-MM-DD <種別>記録を追加`

## 命名規則（再掲）

```
YYYY-MM-DD_<type>_<target>.md
```

例:
- `patch/2026/01/2026-01-18_patch_EC2.md`
- `log/2026/02/2026-02-26_log_check.md`
- `audit/2026-03-04_audit_onsite.md`
- `vulnerability/2025-12-03_vuln_api_access.md`
- `account/2026-01-15_account_request_aws_admin.md`
- `cloudflare/2026-02-10_cf_rule_update.md`
- `incident/2026-01-20_incident_ecs_5xx.md`

## 月次ダイジェスト作成タイミング

- 毎月末（または翌月第1営業日）
- 当月のすべての記録 MD を集めて `MONTHLY/YYYY-MM.md` を作成
- CODIUM へはこの1枚をベースに送る（詳細は MD 内のリンクで辿れる）

## 集計のヒント

フロントマターを利用して下記が機械的に抽出可能:

```bash
# 未対応のパッチ記録を一覧
grep -lR "status: pending" patch/

# 今月のすべての記録
grep -lR "date: 2026-01" .

# 高深刻度の脆弱性のみ
grep -lR "severity: critical\|severity: high" vulnerability/
```

`scripts/` 配下に集計スクリプトを置く予定。
